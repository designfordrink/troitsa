# Троица (Troitsa)

Multi-model decision workflow for AI agents: **Conductor + Worker + Critic**.

Одна задача проходит через три модели с разными ролями:

| Роль | Модель | Что делает |
|------|--------|------------|
| 🎯 **Дирижёр / Conductor** | DeepSeek V4 Pro (OpenRouter) | Думает, планирует, ставит ТЗ, собирает финал |
| ⚡ **Работяга / Worker** | DeepSeek V4 Flash (OpenRouter) | Делает основной объём: анализ, варианты, черновики |
| 🔍 **Критик / Critic** | Owl Alpha (OpenRouter) | Ломает результат работяги, ищет дыры |

Дорогая умная модель только думает и собирает итог, дешёвая делает весь объём, а отдельная независимая модель пытается результат сломать. Так выходит и дешевле, и качественнее, чем одной моделью.

## Ключевой принцип

> **Критик обязан быть другой моделью, чем работяга.**
>
> DeepSeek V4 Flash (Worker) и Owl Alpha (Critic) — разные модели из разных семейств. Если модель проверяет сама себя, она просто соглашается — смысл теряется.

## Установка

Скилл для Hermes Agent:

```bash
hermes skills install "https://raw.githubusercontent.com/designfordrink/troitsa/main/SKILL.md"
```

Или скопировать `SKILL.md` в `~/.hermes/skills/`.

## Использование

В сессии Hermes:

```
/skill troitsa
```

Триггеры: `троица`, `используй троицу`, `прогони через троицу`, `через 3 модели`, `council`, `red team this`

## Когда использовать

Для важных решений по **стратегии, ресёрчу, контенту, нише, продукту или бизнесу**, где цена ошибки высокая.

**НЕ использовать** для: перевод, короткий ответ, поиск ссылки, факт-чек одного утверждения.

## Процесс

1. **Понять задачу** — что реально нужно решить и почему это важно
2. **Критерии** — Conductor формулирует, что считать хорошим ответом
3. **ТЗ для Worker** — Conductor пишет точное задание
4. **Работа** — Worker делает объём (анализ, варианты, черновик)
5. **Критика** — Critic получает результат и пытается его сломать
6. **Правка** — если нужно, 1-2 итерации
7. **Сборка** — Conductor сводит всё в чистый финал
